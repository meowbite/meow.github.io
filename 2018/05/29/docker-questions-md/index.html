docker 分层结构问题汇总

May 29, 2018

文档目的

由于docker镜像的分层结构，会引起一些列的使用上的问题。
镜像分层结构最大的特性是Copy-on-Write：

  • 新数据会直接存放在最上面的容器层。
  • 修改现有数据会先从镜像层数据复制到容器层，修改后的数据直接保存在容器层中，镜像层保持不变。
  • 如果多个层中有命名相同的文件，用户只能看到最上面那层中的文件。

环境信息

  • docker版本： 1.13.1
  • docker storage driver：overlayfs2
  • os版本：centos7.2
  • 内核版本：3.10.0-514.10.1.el7.toa.x86_64
